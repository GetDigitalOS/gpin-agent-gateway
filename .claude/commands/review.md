# Code Review

Review the recent changes in this project. Check against our tier's principles:

1. **Security** — Input validation, no hardcoded secrets, security headers configured
2. **Accessibility** — Semantic HTML, keyboard navigation, color contrast, alt text, focus indicators
3. **Performance** — No unnecessary re-renders, images optimized, lazy loading where appropriate
4. **Code Quality** — DRY, semantic naming, separation of concerns, mobile-first CSS
5. **SEO** — Proper heading hierarchy, meta tags, structured data where applicable

For each issue found, state:
- What the issue is
- Where it is (file and line)
- Why it matters (reference the specific principle)
- How to fix it

Prioritize issues by severity: security > accessibility > correctness > performance > style.
